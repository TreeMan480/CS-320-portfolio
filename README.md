# CS-320-portfolio
Throughout this course, I learned that testing is one of the most important parts of software development. Creating the Contact Service project helped me understand how unit testing can be used to verify that code works correctly and continues to work when changes are made. By writing JUnit tests, I was able to check different scenarios and make sure the program met the project requirements.

How can I ensure that my code, program, or software is functional and secure?

I can ensure my software is functional and secure by testing it throughout the development process instead of waiting until the end. In this course, I used JUnit tests to verify that my classes and methods worked as expected. Testing helped identify errors before they became bigger problems. I also learned the importance of validating input, following requirements carefully, and considering how users might interact with the software. These practices help create programs that are reliable and less vulnerable to unexpected issues.

How do I interpret user needs and incorporate them into a program?

I start by reviewing the project requirements and identifying what the user needs the software to accomplish. From there, I break the requirements into smaller tasks and design the program around those goals. During this course, the project requirements guided the creation of the Contact Service application. Each class and method was developed to satisfy a specific requirement, and testing was used to confirm that those requirements were met. This process helped me understand how user needs drive software design decisions.

How do I approach designing software?

When designing software, I begin by understanding the problem that needs to be solved. Next, I identify the necessary classes, methods, and data needed for the program. I try to keep the design organized and simple so the code is easier to maintain and test. In this course, I used an object-oriented approach to separate responsibilities between classes, which improved readability and organization. I also relied on testing throughout development to verify that the design worked as intended and met the project requirements.

Conclusion

This course strengthened my understanding of software testing, quality assurance, and software design. The projects demonstrated how proper testing can improve software quality and help identify defects early in development. The skills I gained in creating and testing software will be valuable in future courses and professional software development projects.
